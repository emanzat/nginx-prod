You are a senior Java Software Architect

Design a complete enterprise-grade microservice architecture using:

- Java 21
- Spring Boot 4
- PostgreSQL
- Kafka
- Docker
- Kubernetes

The application domain is:

“Government Subsidy Application Management System”

Citizens submit subsidy requests and government agents review, validate, approve or reject them.

The system must be production-ready and follow clean architecture and best practices.

1. ARCHITECTURE

Use a layered architecture:

- Controller
- Service
- Repository
- Entity
- DTO (Model/Structure)
- Exception
- Config
- Security
- Util

Apply best practices:

- Clean architecture
- Separation of concerns
- Centralized exception handling
- Validation

2. SPRING PROFILES

Support multiple Spring profiles:

- dev
- test
- preprod
- prod

Each profile must configure:

- Database
- Logging
- Security
- Kafka

3. DOMAIN MODEL

Entities:

- User
- Application
- Status
- SubsidyType
- Document

Relationships:

- User has many Applications
- Application belongs to User
- Application has Status
- Application has Documents

Status:

- SUBMITTED
- IN_REVIEW
- APPROVED
- REJECTED

4. DTO STRUCTURE

- CreateApplicationRequest

- JWT authentication
- Role-based access control

- ADMIN
- AGENT
- CITIZEN

Authorization:

- ADMIN: full access
- AGENT: review/validate
- CITIZEN: submit/view
